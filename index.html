Selected Projects

1. Hierarchical Control for Bipedal Locomotion

MPC LIPM Dynamics Inverse Kinematics MuJoCo C++

This project focuses on building a bipedal walking controller from scratch. Instead of using black-box RL directly, I adopted a constructive bottom-up approach to validate physical constraints step-by-step.

Phase 1: Inverted Pendulum Model

Phase 1: Simplified Dynamics (LIPM)

Modeled the robot as a Linear Inverted Pendulum to verify balance stability criteria. Ensured the Zero Moment Point (ZMP) remained within the support polygon under disturbance.

Phase 2: CoM Trajectory

Phase 2: CoM Trajectory Planning

Achieved stable Center of Mass (CoM) control along a linear path. Used boundary value solvers to generate smooth trajectories, ensuring the robot could shift weight without falling.

Phase 3: Vertical Leg Control

Phase 3: Vertical Dynamics & IK Validation

Before walking, I focused on vertical leg actuation (squatting and standing). Implemented Inverse Kinematics (IK) to map CoM height changes to joint angles, ensuring feet remained flat and planted while the body moved vertically. This step was crucial to validate torque control capabilities.

Phase 4: Dynamic Walking

Phase 4: MPC-based Dynamic Walking

Integrated Model Predictive Control (MPC) to optimize foot placement in real-time. The robot can now maintain a stable walking gait by actively adjusting step locations to recover from errors.

▶ Deep Dive: Mathematics & Code Implementation
1. Mathematical Derivation

The system dynamics are approximated by the LIPM equation:

$$ \ddot{x}_{com} = \frac{g}{z_c} (x_{com} - p_{zmp}) $$

For Phase 4 (MPC), we solve the following QP problem at 100Hz:

$$ \min_{U} \sum_{k=0}^{N} \| X_{k+1} - X_{ref} \|_Q^2 + \| U_k \|_R^2 $$

Subject to: $X_{k+1} = AX_k + BU_k$ (State Space Dynamics).

2. Scalable Vision System: From Digit to Video

CNN PyTorch OpenCV Real-time Inference

An evolutionary computer vision project. Started with a basic classifier and evolved into a robust system capable of handling video streams.

Stage 1: CNN Classifier

Stage 1: Single Digit Classification

Designed a custom CNN architecture (Conv2D -> ReLU -> MaxPool). Achieved 99.2% accuracy on MNIST test set using data augmentation techniques (Rotation, Scaling).

Stage 2: Sequence Recognition

Stage 2: Multi-Digit Segmentation

Extended the system to recognize sequences (e.g., zip codes). Implemented sliding window and contour detection algorithms to segment connected digits before classification.

Stage 3: Video Stream Analysis

Stage 3: Real-time Video Inference

Optimized the inference pipeline for webcam input. Added temporal consistency checks (moving average of probabilities) to stabilize predictions and reduce flickering in video streams.
