1. Hierarchical Control for Bipedal Locomotion

MPC LIPM Dynamics Inverse Kinematics C++ / MuJoCo

A bottom-up development of a walking controller. I deconstructed the walking problem into four distinct phases, validating the physics and control logic at each step.

Phase 1: Inverted Pendulum (LIPM)

Simplified robot dynamics to a point mass to validate the ZMP stability criterion.

Technical Details: Dynamics Modeling

Equation of Motion: Modeled as a Linear Inverted Pendulum:

$$ \ddot{x} = \frac{g}{z_c}(x - p_{zmp}) $$

To ensure stability without falling, the Zero Moment Point (ZMP) $p_{zmp}$ must satisfy:

$$ p_{min} \leq x - \frac{z_c}{g}\ddot{x} \leq p_{max} $$

Phase 2: CoM Trajectory Planning

Generated smooth CoM trajectories along a linear path to ensure continuous motion.

Technical Details: Boundary Value Problem

Polynomial Interpolation: To ensure continuity in position and velocity between steps, the CoM trajectory $x(t)$ is modeled as a cubic polynomial:

$$ x(t) = a_0 + a_1 t + a_2 t^2 + a_3 t^3 $$

Solved using boundary conditions: $x(0)=x_{start}, \dot{x}(0)=\dot{x}_{start}, x(T)=x_{end}, \dot{x}(T)=\dot{x}_{end}$.

Phase 3: Vertical Leg Control (Squatting)

Implemented Inverse Kinematics (IK) to map CoM height changes to joint angles, validating torque control.

Technical Details: Inverse Kinematics

Geometry: Given the target leg length $L = \sqrt{x^2 + z^2}$, the knee angle $q_{knee}$ is derived using the Law of Cosines:

$$ q_{knee} = \pi - \cos^{-1}\left( \frac{l_1^2 + l_2^2 - L^2}{2 l_1 l_2} \right) $$

Jacobian Mapping: Joint velocities are computed as $\dot{q} = J^{-1} \dot{x}_{task}$.

Phase 4: MPC Dynamic Walking

Integrated Model Predictive Control (QP Solver) for real-time foot placement optimization.

Technical Details: QP Formulation

Optimization Objective: Minimize tracking error and control effort over horizon $N$:

$$ \min_{U} \frac{1}{2} U^T H U + g^T U $$

Constraints:

  • Dynamics: $X_{k+1} = A X_k + B U_k$
  • Stability: $D U_k \leq f$ (ZMP constraints)

2. Scalable Vision System: From Digit to Video

CNN PyTorch OpenCV Real-time Inference

An evolutionary project starting from a basic classifier and scaling to a real-time video analysis system.

Stage 1: CNN Classifier Architecture

Designed and trained a custom CNN on MNIST, achieving 99.2% test accuracy.

Technical Details: Network Structure

Architecture (LeNet-5 Variant):

  • Input: 28x28x1 (Grayscale)
  • Conv1: Kernel=5x5, Out=6, ReLU -> MaxPool2d
  • Conv2: Kernel=5x5, Out=16, ReLU -> MaxPool2d
  • FC1: 120 units -> FC2: 84 units -> Output: 10 units

Stage 3: Video Stream Optimization

Implemented temporal smoothing to reduce flickering in real-time inference.

Technical Details: Temporal Consistency

Exponential Moving Average (EMA):

To prevent prediction flickering between frames, the probability vector $P$ is smoothed over time:

$$ P_{t} = \alpha \cdot P_{raw} + (1 - \alpha) \cdot P_{t-1} $$

Where $\alpha = 0.3$ is the smoothing factor. We only update the displayed label if the confidence exceeds 0.8 for 5 consecutive frames.
